Git to rozproszony system kontroli wersji, który został stworzony przez Linusa Torvaldsa.
System Git pozwala na śledzenie zmian w kodzie źródłowym i współpracę w zespole programistów.
Każdy commit w Git zawiera unikalny identyfikator zwany SHA-1 hash.
Branch w Git to wskaźnik do konkretnego commita w historii projektu.
Merge to operacja łącząca zmiany z różnych branchy w jeden wspólny commit.
Rebase pozwala na przepisanie historii commitów w bardziej liniowy sposób.
Staging area w Git to miejsce gdzie przygotowujemy zmiany przed commitem.
Remote repository to zdalne repozytorium przechowywane na serwerze lub w chmurze.
Pull request to mechanizm proponowania zmian w projekcie open source.
Fork to kopia repozytorium, którą można modyfikować niezależnie od oryginału.
Cherry-pick pozwala na wybranie konkretnego commita z innego brancha.
Stash to tymczasowe przechowanie niezcommitowanych zmian w bezpiecznym miejscu.
Tag w Git służy do oznaczania ważnych wersji projektu, takich jak release.
HEAD to wskaźnik wskazujący na aktualny commit w branchu.
Detached HEAD to stan gdy HEAD nie wskazuje na żaden branch.
Fast-forward merge to sytuacja gdy nie ma konfliktów podczas łączenia branchy.
Conflict resolution to proces rozwiązywania konfliktów podczas merge lub rebase.
Git hooks to skrypty wykonywane automatycznie w określonych momentach.
Gitignore to plik określający które pliki mają być ignorowane przez Git.
Git flow to popularny model workflow dla zarządzania branchami w projektach.
